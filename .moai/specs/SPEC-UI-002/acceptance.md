# SPEC-UI-002 인수 기준: PDF Preview (결과 PDF 미리보기)

## 1. 인수 기준 개요

이 문서는 SPEC-UI-002의 모든 요구사항에 대한 검증 시나리오를 정의한다. Given-When-Then 형식을 사용하며, 각 시나리오는 하나 이상의 요구사항과 연결된다.

---

## 2. 테스트 시나리오

### 시나리오 1: 작업 완료 후 PDF 미리보기 표시 (REQ-U-001, REQ-E-001)

```gherkin
Feature: 작업 완료 후 PDF 미리보기 표시

  Scenario: Cut 작업 완료 후 결과 PDF 미리보기가 표시된다
    Given Cut 작업 페이지에서 PDF 파일이 로드된 상태이다
    And 페이지 범위가 "1-3"으로 입력된 상태이다
    When 사용자가 "실행" 버튼을 클릭하고 작업이 성공적으로 완료된다
    Then 결과 영역에 성공 텍스트 메시지가 표시된다
    And 결과 영역 아래에 결과 PDF의 첫 번째 페이지 썸네일이 표시된다
    And 썸네일 이미지의 크기는 적절한 비율로 리사이즈되어 있다

  Scenario: Merge 작업 완료 후 미리보기가 표시된다
    Given Merge 작업 페이지에 2개 PDF 파일이 추가된 상태이다
    When 사용자가 "실행" 버튼을 클릭하고 병합이 성공적으로 완료된다
    Then 결과 영역에 병합된 PDF의 첫 번째 페이지 썸네일이 표시된다

  Scenario: Rotate 작업 완료 후 회전된 결과를 미리보기로 확인한다
    Given Rotate 작업 페이지에서 PDF가 로드되고 90도 회전이 선택된 상태이다
    When 사용자가 "실행" 버튼을 클릭하고 작업이 완료된다
    Then 결과 영역에 회전된 첫 번째 페이지 썸네일이 표시된다
```

### 시나리오 2: 로딩 인디케이터 표시 (REQ-S-001)

```gherkin
Feature: 미리보기 로딩 상태 표시

  Scenario: 미리보기 렌더링 중 로딩 인디케이터가 표시된다
    Given Cut 작업 페이지에서 PDF 파일이 로드된 상태이다
    When 사용자가 "실행" 버튼을 클릭하고 작업이 성공적으로 완료된다
    Then 미리보기 영역에 로딩 인디케이터가 즉시 표시된다
    And 렌더링이 완료되면 로딩 인디케이터가 썸네일 이미지로 교체된다
```

### 시나리오 3: 렌더링 실패 시 우아한 폴백 (REQ-S-002)

```gherkin
Feature: 미리보기 렌더링 실패 시 폴백

  Scenario: 손상된 PDF 결과의 미리보기 렌더링이 실패한다
    Given 작업이 성공적으로 완료되었으나 결과 PDF가 렌더링할 수 없는 상태이다
    When 시스템이 미리보기 렌더링을 시도한다
    Then 미리보기 영역은 숨겨지거나 빈 상태로 표시된다
    And 텍스트 기반 성공 메시지는 정상적으로 표시된다
    And 에러 메시지나 예외가 사용자에게 노출되지 않는다

  Scenario: pypdfium2가 설치되지 않은 환경에서 실행한다
    Given pypdfium2 라이브러리가 설치되지 않은 환경이다
    When 작업이 성공적으로 완료된다
    Then 미리보기 기능이 자동으로 비활성화된다
    And 기존과 동일하게 텍스트 결과만 표시된다
```

### 시나리오 4: info 페이지 미리보기 제외 (REQ-N-003)

```gherkin
Feature: info 작업은 미리보기를 표시하지 않는다

  Scenario: Info 페이지에서 PDF 정보를 조회한다
    Given Info 작업 페이지에서 PDF 파일이 로드된 상태이다
    When 사용자가 "실행" 버튼을 클릭하고 정보 조회가 완료된다
    Then 결과 영역에 PDF 메타데이터 정보가 표시된다
    And 미리보기 영역은 표시되지 않는다
```

### 시나리오 5: 첫 페이지만 렌더링 (REQ-N-001)

```gherkin
Feature: 첫 번째 페이지만 렌더링한다

  Scenario: 100페이지 PDF의 미리보기
    Given 100페이지짜리 PDF에 대해 작업이 완료된 상태이다
    When 시스템이 미리보기를 렌더링한다
    Then 첫 번째 페이지만 썸네일로 표시된다
    And 추가 페이지는 렌더링되지 않는다
```

### 시나리오 6: UI 차단 없는 렌더링 (REQ-N-002)

```gherkin
Feature: 미리보기 렌더링이 UI를 차단하지 않는다

  Scenario: 대용량 PDF의 미리보기 렌더링 중 UI가 응답한다
    Given 대용량 PDF (50MB+)에 대해 작업이 완료된 상태이다
    When 시스템이 미리보기를 렌더링하는 동안
    Then GUI 윈도우는 정상적으로 응답한다
    And 사이드바 네비게이션이 정상 동작한다
    And 다른 페이지로 전환이 가능하다
```

### 시나리오 7: 미리보기 이미지 클릭으로 PDF 열기 (REQ-E-002, REQ-O-001)

```gherkin
Feature: 미리보기 클릭으로 PDF 열기

  Scenario: 미리보기 썸네일을 클릭하여 전체 PDF를 연다
    Given 작업 완료 후 미리보기 썸네일이 표시된 상태이다
    When 사용자가 미리보기 썸네일을 클릭한다
    Then 시스템 기본 PDF 뷰어에서 해당 PDF 파일이 열린다
```

### 시나리오 8: 새 작업 시작 시 미리보기 초기화 (REQ-C-001)

```gherkin
Feature: 새 작업 시작 시 이전 미리보기 초기화

  Scenario: 새 작업을 시작하면 이전 미리보기가 사라진다
    Given 이전 작업의 미리보기 썸네일이 표시된 상태이다
    When 사용자가 새로운 PDF 파일을 로드하고 작업을 실행한다
    Then 이전 미리보기가 초기화된다
    And 새 작업의 결과에 대한 미리보기가 표시된다

  Scenario: 이전 렌더링 진행 중 새 작업이 시작된다
    Given 이전 작업의 미리보기 렌더링이 진행 중인 상태이다
    When 사용자가 새 작업을 실행한다
    Then 이전 렌더링 결과는 무시된다
    And 새 작업의 결과에 대한 렌더링이 시작된다
```

## 3. 품질 게이트 기준

### 3.1 기능 완성도

| 항목 | 기준 |
|------|------|
| 7개 작업 페이지 미리보기 | Cut, Merge, Split, Rotate, Resize, Compress, Watermark 모두 미리보기 동작 |
| Info 페이지 제외 | Info 작업에서 미리보기가 표시되지 않음 |
| 로딩 인디케이터 | 렌더링 중 로딩 상태가 표시됨 |
| 폴백 처리 | 렌더링 실패 시 텍스트 결과만 정상 표시 |
| 클릭하여 열기 | 썸네일 클릭 시 시스템 뷰어로 PDF 열기 |

### 3.2 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 렌더링 속도 | 일반 PDF (10페이지 이하) 첫 페이지 렌더링 2초 이내 |
| UI 응답성 | 렌더링 중에도 GUI가 멈추지 않음 |
| 메모리 사용량 | 미리보기 표시 시 추가 메모리 50MB 이하 |
| 패키지 크기 | pypdfium2 추가 후 EXE 크기 증가 20MB 이내 |

### 3.3 코드 품질

| 항목 | 기준 |
|------|------|
| 순수 로직 분리 | pdf_preview.py에 GUI 의존성 없음 |
| 타입 힌트 | 모든 공개 함수에 타입 힌트 적용 |
| 에러 핸들링 | 모든 렌더링 실패 경로에서 우아한 폴백 |
| 기존 테스트 통과 | 기존 테스트 스위트 전체 통과 (회귀 없음) |
| 한국어 주석 | 코드 주석이 한국어로 작성됨 |

## 4. 검증 방법

### 4.1 수동 테스트

- 각 작업 페이지(Cut, Merge, Split, Rotate, Resize, Compress, Watermark)에서 작업 실행 후 미리보기 확인
- Info 페이지에서 미리보기가 표시되지 않는 것 확인
- 손상된 PDF로 작업 후 폴백 동작 확인
- 대용량 PDF로 작업 후 UI 응답성 확인
- 미리보기 클릭 시 시스템 뷰어 열기 확인

### 4.2 자동화 테스트

- `pytest`를 이용한 `pdf_preview.py` 단위 테스트
  - 정상 PDF 렌더링 테스트
  - 손상 PDF 에러 핸들링 테스트
  - 빈 PDF (0 페이지) 핸들링 테스트
  - 리사이즈 비율 검증 테스트
- 통합 테스트: `result_display_widget.py` 미리보기 통합 검증

### 4.3 빌드 검증

- PyInstaller 빌드 후 EXE에서 미리보기 기능 동작 확인
- pypdfium2 바이너리가 EXE에 포함되었는지 확인
- EXE 파일 크기 확인
